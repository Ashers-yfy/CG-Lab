## 实验原理

### 质点-弹簧模型

根据胡克定律，两质点 $a$、$b$ 之间的弹力为：

$$f_{a} = -k_{s}(|x_a - x_b| - l)\frac{x_a - x_b}{|x_a - x_b|}$$

阻尼力：

$$f_{d} = -k_{d} v_{a}$$

### 数值积分方法

| 方法 | 速度更新 | 位置更新 | 稳定性 |
|------|----------|----------|--------|
| 显式欧拉 | $v_{t+1} = v_t + a_t \Delta t$ | $x_{t+1} = x_t + v_t \Delta t$ | 差，易爆炸 |
| 半隐式欧拉 | $v_{t+1} = v_t + a_t \Delta t$ | $x_{t+1} = x_t + v_{t+1} \Delta t$ | 较好 |
| 隐式欧拉 | $v_{t+1} = v_t + a_{t+1} \Delta t$ | $x_{t+1} = x_t + v_{t+1} \Delta t$ | 最稳定 |

## 实现说明

### 任务1：场景初始化

将初始化拆分为三个独立 `@ti.kernel`，在 Python 侧顺序调用以保证 GPU 同步：

- `init_positions()`：初始化质点位置、速度，固定顶行两角
- `init_springs()`：建立结构弹簧，记录端点与原长
- `init_spring_indices()`：初始化渲染索引

### 任务2：力学计算

使用 `@ti.func` 声明，编译时内联到调用 kernel，减少 GPU 函数调用开销：

- `compute_forces_on()`：累加重力、阻尼力、弹簧力，使用 `ti.atomic_add` 避免多线程写入冲突
- `clamp_velocity()`：限制最大速度，防止显式欧拉数值爆炸

### 任务3：积分求解器

三个 `@ti.kernel`，每个将受力计算与状态更新合并，最小化 kernel 启动次数：

- `step_explicit()`：显式欧拉，用旧速度更新位置
- `step_semi_implicit()`：半隐式欧拉，先更新速度再更新位置
- `step_implicit_iter()`：隐式欧拉，定点迭代 3 次近似求解

### 任务4：渲染与交互

使用 pygame 渲染，支持：
- 三种积分方法实时切换
- 阻尼系数实时调节
- 暂停 / 重置功能
- 键盘快捷键（1/2/3 切换，Space 暂停，R 重置）
